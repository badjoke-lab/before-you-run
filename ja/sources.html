Tripwireのソース区分

情報の確度を見分けるため、ソースを3段階で扱います。

Primary

公式アドバイザリや公開元の一次情報。確度は高めです。

Reference

信頼できる第三者の解説。一次情報との照合が前提です。

Signal

初期報告や観測情報。未確定のため事実確定とは分けて扱います。

候補収集に使う情報源

Tripwireは以下の情報源を候補収集用の対象として使います。収集された情報は自動公開されず、カード化する前に必ずレビューします。

Primary / 公式・プロジェクト系

  • CISA Advisories
  • PyPI Blog
  • Node.js Blog
  • Ruby News
  • Rust Blog

Reference / エコシステム系

  • GitHub Security Blog
  • GitHub Changelog
  • OpenSSF Blog

その他のシグナル

SNS、コミュニティ投稿、スクリーンショット、手動で渡されたURLは補助情報として扱います。それ単体では確定情報として扱いません。
